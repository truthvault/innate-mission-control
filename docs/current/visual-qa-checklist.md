# Visual QA Checklist

Use this before telling Guido a page is ready.

## Required Views

- Desktop first viewport.
- Tablet or intermediate-width first viewport. Do not skip the awkward in-between widths.
- Mobile first viewport.
- One mid-page section, especially the most complex section.

## Check These

- Header, announcement bar, nav and hero feel calm together.
- H1 size fits the page type.
- H2s are not louder than the page title.
- No awkward heading wraps or single-word tail lines.
- Images are real, useful and cropped properly.
- Image zoom/lightbox popups have top breathing room, a visible close button inside the image, no unnecessary frame, and no mobile overflow.
- Lightbox labels and arrows only appear for like-sized image sets; smaller swatches stay single-image so the popup does not jump sizes.
- Buttons are clear, aligned and not over-styled.
- Forms have visible fields and strong focus states.
- Cards are clean and not nested.
- Accordions match the dining standard.
- Google Reviews match the homepage style.
- Section spacing has no dead zones.
- No horizontal overflow on mobile.
- No raw app/widget styling.
- No key information was removed just to make the page cleaner.
- Main text, images, links and repeated blocks remain editable in Shopify where practical.
- Handoff includes a plain-English Shopify editor overview for the updated page.
- New live images are not dependent on staging theme asset URLs.

## If Visual QA Is Blocked

Say the page is staged but not visually cleared. Do not replace visual inspection with code confidence.
